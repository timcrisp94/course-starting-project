Core Types

  • Number: all numbers, same as JS
  • String: all text values, same as JS
  • Boolean: true, false; no truthy falsy values
  • Object: any JS object, and more specific types
  • Array: any JS array, and more specific types
  • Tuple: not recognized in JS, recall a tuple is an array with a fixed length
  • enum: {NEW, OLD}
  • any: * (avoid)

TypeScript does not change run-time js, it helps us through development

--TypeScript is static while JS is dynamic

Assigning types

Type inference: TS does its best to understand type by looking at the variable assignment. If a variable is assigned a value, TS gets it. However, if you declare a variable without setting a value, it is good practice to assign a type. For example:

let number1: number

Union Types

A union type describes a value that can be one of several types. We use the vertical bar (|) to separate each type, so number | string | boolean is the type of a value that can be a number, a string, or a boolean.

Literal Types

A literal is a more concrete sub-type of a collective type. There are three sets of literal types available in TypeScript today: strings, numbers, and booleans; by using literal types you can allow an exact value which a string, number, or boolean must have. In practice string literal types combine nicely with union types, type guards, and type aliases. You can use these features together to get enum-like behavior with strings.

Type Aliases

A type alias is exactly that - a name for any type. Aliases are only that--aliases. You cannot use type aliases to different "versions" of the same type. You create aliases with the type keyword
